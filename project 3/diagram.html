How the data is organised in recipes.xml

<recipes>
<recipe id="1"> ×4 id attr
<title>
Jollof Rice
<category>
Main Course
<cookingtime>
45 minutes
<ingredients>
<item>
2 cups rice
<item> ×n
<steps>
<step>

<step> ×n

🔑 Legend

Root element — wraps all recipes
Repeating element — one per recipe (×4), has an id attribute
Child element — one piece of recipe info
Nested child — repeats for each ingredient / step
Example value stored inside the element
